## Methodology

All 29 icons were generated by a single Python script (`scripts/generate_method_a_icons.py`) that constructs SVG XML strings directly (no external library). Each icon is a Python function returning an SVG string. The script writes files and records execution timing.

**Script execution time** (writing files to disk) is sub-millisecond per icon. The meaningful time investment is **design + coding**: planning SVG path geometry for each icon and writing the Python functions. This is tracked per icon below as "design time" — the wall-clock time the agent spent reasoning about and coding each icon function.

## Style Guide Deviations

| Icon | Deviation | Reason |
|------|-----------|--------|
| `icon_item_module_*` (all 3) | Front-face detail elements use `stroke-width="1.5"` instead of `2` | Detail symbols (recycler arrow, fabricator gear, hub network) are drawn at smaller scale on the isometric box's front face. Using stroke-width 2 made detail lines merge with the box outline at small sizes. The box outline itself uses the standard stroke-width 2. |
| `icon_hud_drone` | Rotor circles use `stroke-width="1.5"` | Four rotor discs at corners need thinner strokes to avoid merging with arm lines at 24px display. Central body circle uses standard stroke-width 2. |
| `icon_hud_heat` | Thermometer bulb uses `fill="currentColor"` | The style guide specifies `fill="none"` for HUD icons but the icon list notes the bulb "can use `fill='currentColor'` for visual clarity." Followed the icon list recommendation. |

No icons were skipped. All 29 icons from the TICKET-0086 audit are produced.
