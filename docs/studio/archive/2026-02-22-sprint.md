## Process Events

### Process Violation — TICKET-0015
technical-artist proceeded to complete TICKET-0012 and TICKET-0013 while TICKET-0011 was `IN_REVIEW` awaiting Studio Head approval. TICKET-0015 (BLOCKER) was created to gate QA until the pipeline decision was formally approved.

**Root cause:** `depends_on` semantics were undefined in the ticket system — agents could read a dependency as satisfied when it was `IN_REVIEW` rather than `DONE`.

**Resolution:** Studio Head approved the hybrid pipeline recommendation. technical-artist's Blender-only asset selection (TICKET-0013) was rejected; TICKET-0016 reworked the asset set per the approved hybrid pipeline.

**Process improvements shipped:**
- `tickets/README.md`: `depends_on` semantics explicitly defined — only `DONE` clears a dependency
- `agents/_template/CLAUDE.md`: Mandatory prerequisite check added to all agent "Receiving Work" protocols
- `agents/technical-artist/CLAUDE.md`: Same prerequisite check + guardrail against unilateral deviation from approved pipeline decisions
- `agents/qa-engineer/CLAUDE.md`: All findings (P0–P3) must be logged in QA ticket Activity Log before sign-off
