## Scope Delivered

### Gamepad Input Remapping

- **Interact → X button** (TICKET-0276): `interact` action reassigned from `JOY_BUTTON_A` to `JOY_BUTTON_X`. UI audit confirmed no hardcoded gamepad button labels in any `.gd` or `.tscn` file — all labels dynamically resolved via `InputMap.action_get_events()`.
- **Jump → A button, Ping → LB** (TICKET-0277): `JOY_BUTTON_A` bound to `jump`; `JOY_BUTTON_LEFT_SHOULDER` bound to `ping`. Input action renamed from `"scan"` to `"ping"` across all files — `scanner.gd`, `interaction_prompt_hud.gd`, `test_input_manager_unit.gd`, and all call sites updated.
- **Use Item → A button, Headlamp → RB** (TICKET-0278): `JOY_BUTTON_A` added to `use_item`; `JOY_BUTTON_RIGHT_SHOULDER` added to `toggle_head_lamp`. Context separation confirmed: `set_gameplay_inputs_enabled(false)` suppresses `jump` and `use_item` when inventory UI is open.
- **Mining → RT (Right Trigger)** (TICKET-0279): New `_add_joy_axis_to_existing_action()` helper added to `InputManager.gd` to handle `InputEventJoypadMotion` events. `JOY_AXIS_TRIGGER_RIGHT` mapped to `use_tool` at 0.5 threshold. `interaction_prompt_hud.gd` updated with `_joy_axis_name()` to display "RT"/"LT" labels.

### Ship Boarding Feel

- **Ship boarding raycast** (TICKET-0280): `_is_aiming_at_ship()` method added to `DebugShipBoardingHandler`. Camera-forward raycast against `PhysicsLayers.ENVIRONMENT` verifies the hit collider is a descendant of a "ship" group node. Ray length 30 m. Boarding now only triggers when player points at ship hull. Graceful degradation to proximity-only if camera ref is null. Resolves D-034.

### Scanner Improvements

- **Scanner radial wheel** (TICKET-0281): `resource_type_wheel.gd` created — radial wheel Control that builds segments dynamically from `ResourceDefs.RESOURCE_CATALOG`. Hold ping (Q/LB) opens wheel; release fires type-filtered ping. Tap fires with last-used type. Mouse direction / left stick selects segment. Scanner's `_do_ping()` now filters deposits by resource type. Resolves D-001.
- **Animated ping propagation ring** (TICKET-0282): Frame-driven ring propagation system added to `scanner.gd` — `PING_SPEED` 100 m/s, `PING_RANGE` expanded to 1000 m. New `deposit_ping_revealed` signal emits per-deposit as ring front reaches each one. `game_hud.gd` connects to `deposit_ping_revealed` for progressive compass marker reveal. `ping_completed` still fires immediately with full list — no downstream regressions. Resolves D-015.

### Orchestrator Tooling

- **--max-turns replaces budget_usd** (TICKET-0283): `get_budget()` removed from `conductor.py`. `budget_usd` fields removed from `orchestrator/config.json`. New `max_turns` config section: default=75, gameplay-programmer=150, qa-engineer=100, producer=25. All dispatch paths unconditionally pass `--max-turns <agent_max_turns>`. Zero orphaned budget/get_budget references remain. Resolves D-033.

### Producer Decisions

- **D-007 requirements confirmed** (TICKET-0284): Studio Head interview recorded in-biome timed respawn requirements. Per-node timer (not per-biome), per-resource-type config at 300 s default, deep nodes excluded, no scarcity pressure goal. TICKET-0286 created and completed for implementation. D-007 updated in `deferred-items.md`.

### Resource Gameplay

- **In-biome timed resource node respawn** (TICKET-0286): `ResourceRespawnConfig` created in `game/config/resource_respawn_config.gd` with `RESPAWN_TIMES` dict and `get_respawn_time()` static helper (default 300 s for all types). `deposit.gd` modified to start a one-shot `Timer` on full depletion — hides mesh, disables collision, restores full yield on expiry. Deep nodes (`infinite = true`) excluded. Depleted nodes already excluded from scanner/compass via existing `is_depleted()` filters — no scanner changes needed. Resolves D-007.

### P3 Finding — test_debris_field_biome_unit OOM in Headless Mode

- **Date:** 2026-03-03 [qa-engineer] FINDING [P3]: test_debris_field_biome_unit — Headless test run crashes with "Out of memory" during `_assemble_full_mesh()` in `terrain_generator.gd`. Signal 4 (SIGILL) from OS memory allocator failure. Disposition: **known issue, acceptable for milestone** — this is a pre-existing headless infrastructure limitation, not a M10 regression. `terrain_generator.gd` last modified in TICKET-0162 (M8); `test_debris_field_biome_unit.gd` last modified in TICKET-0176 (M8 gameplay phase gate). Neither file was touched in M10. M9 QA confirmed this test passed in the Godot editor environment (M9 Phase Gate Summary: "879/879 PASS"). Mesh assembly for a full 500 m × 500 m terrain requires GPU memory unavailable in headless rendering mode. Recommended fix: add a `--headless` guard in the test's `before_each()` or mock terrain mesh generation; deferred to future sprint.
