## Summary

Render auto-generated dependency graph diagrams on the dashboard, using Mermaid definitions produced by the build script (TICKET-0190) from ticket `depends_on` and `blocks` fields.

## Acceptance Criteria

### Diagram Rendering
- [ ] Each milestone detail view includes a dependency graph diagram
- [ ] Diagram rendered via Mermaid.js using the `.mmd` definitions generated by the build script
- [ ] Nodes represent tickets (labeled with ticket ID and abbreviated title)
- [ ] Edges represent dependency relationships (arrow from dependency to dependent ticket)
- [ ] Node color indicates status: green (DONE), yellow (IN_PROGRESS), grey (OPEN)

### Layout and Readability
- [ ] Diagram uses a top-down (`TD`) or left-right (`LR`) flow direction
- [ ] Readable for milestones with 5–30 tickets (no overlapping nodes)
- [ ] Orphan tickets (no dependencies, not depended on) shown in a separate cluster or at the top
- [ ] Diagram fits within the page width (scrollable if very large)

### Edge Cases
- [ ] Milestones with no dependencies show a flat list of nodes (no edges)
- [ ] Cross-milestone dependencies shown with a different edge style (dashed line) and labeled with source milestone
- [ ] Circular dependencies (if any) flagged visually (red node border) rather than crashing Mermaid

### Integration
- [ ] Diagram appears below the ticket table on each milestone detail page
- [ ] Collapsible via a toggle ("Show/Hide Dependency Graph")
- [ ] Mermaid re-renders correctly when navigating between milestones

## Implementation Notes

- The build script (TICKET-0190) generates `.mmd` files in `dashboard/dist/data/diagrams/`. The dashboard JS loads these and passes them to Mermaid for rendering.
- For large milestones (M5 has 38 tickets), the diagram may need pagination or a simplified view. Start with the full graph and add simplification if needed.
- Mermaid `flowchart` syntax is recommended over `graph` for better control over node styling.
